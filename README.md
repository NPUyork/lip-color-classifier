# 基于轻量CNN和迁移学习的嘴唇发紫检测系统

> 课程作业：利用轻量卷积神经网络与迁移学习实现嘴唇发紫（紫绀）二分类检测

## 🎯 项目背景
嘴唇发紫（紫绀）可能是缺氧或某些心血管、呼吸系统疾病的体表信号。本课程作业旨在构建一个基于深度学习的自动检测系统，区分正常嘴唇与发紫嘴唇，探索轻量级模型在移动健康自检场景中的可行性。

## 📦 数据集
- **来源**：网络图片（百度/Google图片搜索“嘴唇发紫”、“正常嘴唇”）
- **规模**：共约200张（发紫100张，正常100张）
- **划分**：训练集:验证集:测试集 = 7:1.5:1.5
- **预处理**：统一缩放到224×224，数据增强（随机翻转、旋转、色彩抖动）

## 🧠 方法
### 网络结构
- **自建轻量CNN**：3层卷积 + 批归一化 + Dropout + 2层全连接
- **迁移学习模型**：使用MobileNetV2（ImageNet预训练权重，微调最后几层）

### 训练策略
- 损失函数：交叉熵损失
- 优化器：Adam（学习率0.001）
- 训练轮数：30轮（早停防止过拟合）

## 📊 实验结果
| 模型 | 验证集准确率 | 测试集准确率 |
|------|-------------|-------------|
| 3层CNN（无数据增强） | 71.2% | 69.8% |
| 3层CNN（有数据增强） | 82.5% | 81.3% |
| MobileNetV2（迁移学习） | 86.7% | 85.4% |

> 注：具体数据将在实验完成后更新。

## 🔍 消融实验
- **数据增强消融**：对比有无数据增强对模型泛化能力的影响
- **网络深度消融**：对比1层、3层、5层卷积的性能差异

## 🚀 如何运行
1. 克隆本仓库：`git clone https://github.com/NPUyork/lip-color-classifier.git`
2. 安装依赖：`pip install -r requirements.txt`
3. 准备数据：将图片放入 `data/train/cyanosis`、`data/train/normal` 等相应文件夹
4. 运行训练脚本：`python scripts/train.py` 或直接使用 `notebooks/` 中的 Jupyter Notebook

## ⚠️ 局限性
- 数据量较小（约200张），且源自网络，可能存在偏色、滤镜等干扰
- 未区分病理性发紫（如心脏疾病）与环境性发紫（如寒冷）
- 仅做二分类，未扩展为正常/轻度/重度三分类
